# wikiSearch
wikiSearch is a distributed web application developed in fulfilment of CT5169 (Fundamentals of Cloud Computing, University of Galway). It uses Flask, an EC2-hosted Python script for Wikipedia queries, and a MySQL Docker container for caching.

## Implementation and Setup
The web application was developed in python using flask framework and GitHub as version control. 
Three virtual machines where created. VM1 and VM2 running locally while an amazon EC2 instance (Amazon Web Services, n.d.) running in the cloud with the below specifications:
> - Type: Linux
> - Version: Ubuntu 64 bits
> - Memory Size: 2048Mb
> - Processors: 2
> - File Size: at least 25Gb

### Amazon EC2 Instance:
#### Setup:
SSH into the EC2 instance :  
>`chmod 400 mykey.pem` <br>
>`ssh -i /mykey.pem ubuntu@<EC2_PUBLIC_IP >`

update and install python3-venv this installs a virtual environment for python :  
>`sudo apt update`  
>`sudo apt install python3-venv -y`


create a virtual environment called wiki_env and activate it:  
>`python3 -m venv wiki_env`  
>`source ~/wiki_env/bin/activate`  

then install Wikipedia inside the virtual environment:  
>`pip install wikipedia`

 A file wiki.py was created and the python script modified and copied into it:  
 >``nano wiki.py``  
```python
import wikipedia
import sys

def searchWikipedia(term):
    pageIds = wikipedia.search(term)
    pages = []
    for pageId in pageIds:
        try:
            page = wikipedia.page(pageId)
            pages.append(page)
        except:
            print(f"Oups: could not parse the page: {pageId}")
            pass

    if len(pages) > 0:
        for page in pages:
            print("title: " + page.title)
            print("URL: " + page.url)
            print("Content: " + page.content[:500])
            print("-" * 50)
    else:
        print("No results!")

if __name__ == "__main__":
    arguments = " ".join(sys.argv[1:])
    searchWikipedia(arguments)
```

### VM1: 
Docker was used to containerize the MySQL database in this VM (Docker Inc., n.d.), The MySQL database was is used in caching search terms and results that were fetched from Wikipedia, improving performance and reducing repeated API calls.
#### Setup:
Forwarding port name: MySQL, TCP, Host port: 7888, Guest port: 6603  

Install docker:  
>``sudo apt update``  
>``sudo apt install docker.io -y``  
>``sudo systemctl start docker``  
>``sudo systemctl enable docker``  
>``sudo docker pull mysql:latest``  

and run the MySQL in a container in the background  
```bash
sudo docker run -d \
  --name mysqlcontainer1 \
  -e MYSQL_ROOT_PASSWORD=<yourpassword> \
  -e MYSQL_DATABASE=cache_db \
  -p 6603:3306 \
  mysql:latest
```
to create the cache table do.  
>``docker exec -it mysqlcontainer1 mysql -u root -p``  

enter MySQL password and  
```sql
CREATE DATABASE IF NOT EXISTS cache_db;
USE cache_db;

CREATE TABLE cache (
    id INT AUTO_INCREMENT PRIMARY KEY,
    query VARCHAR(255),
    result TEXT
);
```
The first time u make a search it will take a while but if you reapeat same search you will notice a faster improvement.
